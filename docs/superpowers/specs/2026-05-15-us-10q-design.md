# 美股 10-Q 季报补齐 — 设计

## 背景

SEC EDGAR 只支持 10-K 年报，无 10-Q 季报。CLI `reports` 命令硬限制仅 A 股。

## 设计

### 1. EDGAR Provider (`src/data/providers/edgar.py`)

**重构为通用 filings 接口：**

| 旧 | 新 |
|----|----|
| `list_10k(ticker)` | `list_filings(ticker, form_types=None, since_year=None)` |
| `download_10k(ticker, data_dir)` | `download_filings(ticker, since_year=None, form_types=None)` |

- `form_types` 默认 `["10-K", "10-Q"]`
- 标题中通过 `10-K` / `10-Q` 关键字过滤
- 复用已有 CIK 解析、API 调用、文本下载逻辑
- 文本保留上限 500KB，0.2s 间隔遵守 SEC 速率限制

`report_type` 映射：`"10-K"` → `"annual"`, `"10-Q"` → `"quarterly"`

### 2. Gateway (`src/data/gateway.py`)

`get_reports` 美股分支透传 `since_year` / `report_types`：

```python
return self._edgar.download_filings(symbol, since_year=since_year, form_types=form_types)
```

### 3. CLI `fetch reports`

- 去掉 `market.value != "cn"` 硬限制
- US 市场：`report_type` 映射 `annual` → `["10-K"]`, `quarterly` → `["10-Q"]`, `all` → `["10-K", "10-Q"]`
- `--years` 默认近 2 年同 A 股逻辑

### 4. 报告文本分析 (`src/analysis/report_text.py`)

- `_find_latest_report` 加 US 路径查找
- SEC 文本格式不同（无审计意见章节），简化分析：仅提取风险关键词 + 管理层展望

### 5. 存储路径

`data/stock/us/{ticker}/reports/{YYYY}_{10K|10Q}_{ticker}.txt` — 已有路径，不变。

### 6. 不做的

- 不新增 Provider
- 不改存储结构
- 不新增 SEC 速率限制配置（已内联 `time.sleep(0.2)`）

## 受影响文件

| 文件 | 改动 |
|------|------|
| `src/data/providers/edgar.py` | `list_10k`→`list_filings`, `download_10k`→`download_filings` |
| `src/data/gateway.py` | `get_reports` US 分支透传新参数 |
| `src/cli/commands/reports.py` | 去 A 股限制，加 US 映射 |
| `src/analysis/report_text.py` | 加 US 路径查找 |
